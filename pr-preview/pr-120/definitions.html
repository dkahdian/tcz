Core Concepts

Representation Language

A representation language LLL is a set of Boolean formulas sharing a common syntactic structure, endowed with the standard Boolean semantics and a size measure ∣⋅∣|\cdot|∣⋅∣, usually simply the count of all variables and connectives in the formula.

Succinctness Between Languages

There are two intuitive definitions of succinctness: a time requirement, i.e. that an efficient compilation algorithm exists between two languages; or the space requirement, that formulas remain ``small'' in the new language. Since, for any computation, runtime is lower-bounded by output size, the latter definition is weaker.

We fix the convention that succinctness between representation languages is a size-only relation, with no requirement that a conversion algorithm exist . However, to the best of our knowledge, all known proofs of succinctness use a compilation algorithm to establish the size relationship; and all proofs that no time-efficient compilation algorithm exists use space complexity bounds. An example to the contrary would be highly interesting.

Formally, let L1L_1L1​ and L2L_2L2​ be two languages. L1L_1L1​ is at least as succinct as L2L_2L2​, written L1≤pL2L_1 \leq_p L_2L1​≤p​L2​, if there exists a polynomial ppp such that for every α∈L2\alpha \in L_2α∈L2​ there exists an equivalent β∈L1\beta \in L_1β∈L1​ with ∣β∣≤p(∣α∣)|\beta| \leq p(|\alpha|)∣β∣≤p(∣α∣).

We say L1L_1L1​ polynomially compiles into L2L_2L2​ if there exists a polynomial-time algorithm that converts any α∈L1\alpha \in L_1α∈L1​ into an equivalent β∈L2\beta \in L_2β∈L2​.

Since all claims in this project work with either definition, we will sometimes use them as if they were interchangeable.

We additionally define quasi-polynomial succinctness identically, except we require a quasi-polynomial function q(n)∈nlogO(1)(n)q(n) \in n^{log^{O(1)}(n)}q(n)∈nlogO(1)(n) satisfying the above properties and use the notation ≤q\leq_q≤q​.

Tractability of an Operation

An operation O\mathcal{O}O is tractable on a representation language LLL if there exists a polynomial-time (resp. quasi-polynomial-time) algorithm that performs O\mathcal{O}O on any input formula(s) in LLL. Following , the point is to compile a propositional theory off-line into a language where target operations are tractable.

Queries

A query takes a formula α∈L\alpha \in Lα∈L (and possibly auxiliary inputs) and returns a Boolean answer or a numerical value without modifying α\alphaα.

The standard query operations in this project are Consistency (CO), Validity (VA), Clausal Entailment (CE), Implicant (IM), Equivalence (EQ), Model Counting (CT), Model Enumeration (ME), and Sentential Entailment (SE).

Consistency (CO)

Given a sentence α\alphaα, decide whether α\alphaα is satisfiable, i.e., whether there exists an assignment XXX such that α(X)=⊤\alpha(X)=\topα(X)=⊤.

Validity (VA)

Given a sentence α\alphaα, decide whether α\alphaα is valid, i.e., α≡⊤\alpha \equiv \topα≡⊤.

Clausal Entailment (CE)

Given a sentence α\alphaα and a clause ccc, decide whether α⊨c\alpha \models cα⊨c (equivalently, whether α∧¬c\alpha \wedge \neg cα∧¬c is unsatisfiable).

Implicant (IM)

Given a sentence α\alphaα and a term ttt, decide whether t⊨αt \models \alphat⊨α (equivalently, whether t∧¬αt \wedge \neg \alphat∧¬α is unsatisfiable).

Equivalence (EQ)

Given two sentences α\alphaα and β\betaβ, decide whether α≡β\alpha \equiv \betaα≡β.

Model Counting (CT)

Given a sentence α\alphaα, compute the number of satisfying assignments: ∣{X:α(X)=⊤}∣|\{X:\alpha(X)=\top\}|∣{X:α(X)=⊤}∣.

Model Enumeration (ME)

Given a sentence α\alphaα, enumerate all satisfying assignments, i.e., output {X:α(X)=⊤}\{X:\alpha(X)=\top\}{X:α(X)=⊤}.

Sentential Entailment (SE)

Given sentences α\alphaα and β\betaβ, decide whether α⊨β\alpha \models \betaα⊨β.
